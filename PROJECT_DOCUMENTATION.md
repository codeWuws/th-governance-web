# 医学数据治理质控平台 - 项目文档

## 📋 项目概述

医学数据治理质控平台是一个面向医学科研的数据治理和质控系统，用于完成对医学数据的质控和治理工作。系统采用 React 19 + TypeScript + Vite + Ant Design 5 + SCSS 技术栈构建，提供完整的数据治理、质控、管理和检索功能。

### 技术栈

- **前端框架**: React 19
- **开发语言**: TypeScript
- **构建工具**: Vite
- **UI 组件库**: Ant Design 5
- **样式方案**: SCSS（模块化）
- **状态管理**: Redux Toolkit
- **路由管理**: React Router v7
- **数据可视化**: ECharts
- **HTTP 客户端**: Axios

## 📦 功能模块详解

### 1. 仪表盘 (Dashboard)

**路由路径**: `/dashboard` 或 `/`

**功能描述**: 数据治理工作流的概览和监控中心

**主要功能**:

- **工作流执行统计**

  - 总工作流数统计
  - 运行中工作流数量
  - 已完成工作流数量
  - 执行失败工作流数量
- **工作流健康度指标**

  - 工作流成功率可视化（进度条展示）
  - 工作流步骤配置状态（已启用/已禁用比例）
- **工作流步骤配置预览**

  - 显示前 5 个工作流步骤
  - 展示步骤名称、启用状态、自动执行标识
  - 支持跳转到完整配置页面
- **最近执行记录**

  - 显示最近 5 条执行记录
  - 展示批次名称、执行状态、开始/结束时间
  - 支持点击跳转到工作流详情页

**交互功能**:

- 点击"启动工作流"按钮 → 跳转到工作流配置页面
- 点击"工作流配置"按钮 → 跳转到工作流配置页面
- 点击"执行历史"按钮 → 跳转到执行历史页面
- 点击执行记录 → 跳转到对应工作流详情页
- 支持刷新最近执行记录

### 2. 数据库连接管理 (DatabaseConnection)

**路由路径**: `/database-connection`

**功能描述**: 管理系统与外部数据库的连接配置

**主要功能**:

- **连接列表管理**

  - 搜索建议（自动补全）
  - 热门关键词推荐
  - 搜索历史记录
- **搜索结果展示**

  - 分页显示搜索结果
  - 高亮显示匹配关键词
  - 显示搜索耗时和结果总数
  - 结果排序（相关性/时间）

  - 热门关键词趋势

**交互功能**:

- 点击热门关键词 → 使用该关键词搜索
- 点击结果记录 → 查看详细信息或跳转到可视化页面

#### 7.2 高级检索 (AdvancedSearch)

**路由路径**: `/data-retrieval/advanced`

**功能描述**: 基于多条件组合的高级检索

**主要功能**:

- **条件构建**

  - 条件逻辑关系（AND/OR）
  - 条件值类型（文本/数字/日期/枚举）

  - 执行高级检索
  - 保存检索条件
  - 加载已保存的检索条件
- **结果展示**

  - 分页显示结果
  - 结果导出
  - 跳转到可视化分析

**交互功能**:

- 添加检索条件 → 选择字段、操作符、输入值
- 设置条件关系 → 选择 AND/OR
- 执行检索 → 显示结果

## 🎯 核心功能交互流程

### 1. 数据治理工作流执行流程

```
1. 进入仪表盘 → 查看工作流统计
2. 点击"启动工作流" → 跳转到工作流配置页
3. 配置工作流步骤（启用/禁用、设置参数）
4. 点击"启动工作流" → 打开启动对话框
5. 输入工作流名称 → 确认启动
6. 系统创建执行任务 → 自动跳转到工作流详情页
7. 实时查看执行进度 → 各步骤状态更新
8. 执行完成 → 查看执行结果和日志
9. 返回执行历史 → 查看所有历史记录
```

### 2. 数据质控流程

```
1. 进入数据质控首页 → 选择"综合质控管理"
2. 选择质控类型标签页（可靠性/及时性/完整性/一致性/准确性）
3. 点击"启动质控流程" → 打开启动对话框
4. 选择质控类型（可多选） → 输入流程名称
5. 确认启动 → 系统创建质控任务
6. 自动跳转到质控流程详情页
7. 实时查看各质控步骤执行状态
8. 查看质控结果 → 点击步骤查看详细结果
9. 导出质控报告（如需要）
```

### 3. 数据检索流程

```
1. 进入数据检索模块 → 选择检索方式（全文/高级/条件树）
2. 全文检索：
   - 输入关键词 → 自动搜索（防抖）
   - 查看搜索结果 → 分页浏览
   - 点击结果 → 查看详情或跳转可视化
3. 高级检索：
   - 添加检索条件 → 设置字段、操作符、值
   - 设置条件关系（AND/OR）
   - 执行检索 → 查看结果
   - 保存检索条件（可选）
4. 条件树检索：
   - 构建条件树 → 添加条件组和条件项
   - 执行检索 → 查看结果
5. 跳转到可视化视图 → 查看数据可视化
6. 跳转到检索分析 → 进行统计分析
```

### 4. 数据资产管理流程

```
1. 进入数据管理 → 选择"数据资产管理"
2. 浏览树形结构 → 数据源 → 资产类别 → 表 → 字段
3. 点击树节点 → 右侧显示详细信息
4. 新增数据源：
   - 点击"新增资产" → 填写数据源信息
   - 保存 → 树形结构更新
5. 新增资产类别：
   - 选择数据源 → 右键或点击按钮
   - 填写类别信息 → 保存
6. 查看表和字段：
   - 展开到表节点 → 查看字段列表
   - 点击字段 → 查看字段详情
```

## 📝 开发规范

### 代码规范

1. **组件开发**

   - 使用函数组件和 TypeScript
   - 组件文件放在对应模块的文件夹下
   - 样式文件使用 SCSS 模块化（`.module.scss`）
2. **服务层**

   - 每个模块的 API 请求写在对应的 service 文件中
   - Service 文件放在 `src/services/` 目录
3. **类型定义**

   - 所有类型定义放在 `src/types/` 目录
   - 使用 TypeScript 严格模式
4. **工具函数**

   - 通用工具函数放在 `src/utils/` 目录
   - 模块特定工具放在对应模块的 `utils/` 文件夹

### 样式规范

1. **SCSS 模块化**

   - 每个组件使用独立的 `.module.scss` 文件
   - 使用 CSS Modules 避免样式冲突
2. **全局样式**

   - 全局变量定义在 `src/styles/variables.scss`
   - 全局混入定义在 `src/styles/mixins.scss`
   - 全局样式定义在 `src/styles/global.scss`

### 交互规范

1. **弹窗使用**

   - 优先使用封装好的 `showDialog` 函数
   - 自定义 Dialog 使用 `CustomDialog` 组件
   - 新增 Dialog 组件放在 `components/` 目录下新建文件夹
2. **消息提示**

   - 使用 `uiMessage` 工具进行消息提示
   - 成功/错误/警告/信息使用对应方法
3. **确认操作**

   - 使用 `showConfirm` 工具进行确认对话框
   - 删除等危险操作必须使用确认对话框
