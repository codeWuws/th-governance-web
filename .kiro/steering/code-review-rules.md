# 代码审查专用规则

当你在聊天中使用 `#code-review-rules` 时，我会应用这些规则：

## 代码审查检查清单
- 检查 TypeScript 类型安全性
- 验证错误处理的完整性
- 确认组件的可复用性
- 检查性能优化点
- 验证无障碍性支持
- 确认国际化支持

## 医学数据特殊审查
- 验证数据脱敏处理
- 检查医学术语使用的准确性
- 确认数据安全合规性
- 验证审计日志记录

## 代码质量标准
- 函数复杂度不超过 10
- 组件 props 必须有完整的 TypeScript 接口
- 所有异步操作必须有错误处理
- 关键业务逻辑必须有单元测试
