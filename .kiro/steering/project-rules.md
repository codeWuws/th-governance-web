# 医学数据治理平台 - Kiro 开发规则

## 项目概述
这是一个医学数据治理质控平台，用于医学科研的数据治理和质控工作。项目使用 React 19 + TypeScript + Vite + Ant Design 5 + SCSS 技术栈。

## 核心开发约束

### 代码质量要求
- 代码必须生产就绪，包含完整的错误处理
- 遵循最佳实践和设计模式
- 添加清晰的中文注释说明复杂逻辑
- 考虑性能优化和边界情况
- 严格遵循 TypeScript 类型规范

### 技术栈规范
- 使用 React 19 函数组件，禁用废弃的 API
- 使用 Ant Design 5 最新语法，避免废弃写法
- 使用 SCSS 模块化样式编写 (`.module.scss`)
- 每个页面都是一个文件夹，包含组件、样式、mock 数据

### 架构规范
- 模块化开发，不同模块的 API 请求写在对应的 service 中
- 页面风格要求简约优雅，保持高度统一
- 优先查看系统内是否有相同模块或封装的函数，避免重复造轮子

### 弹窗和交互规范
- 弹窗优先使用封装好的 `showDialog` 函数
- 自定义弹窗使用 `CustomDialog` 组件
- 新增 Dialog 组件放入 `components/` 目录下新建文件夹，命名见名知意
- 使用 `uiMessage` 进行消息提示
- 危险操作必须使用 `showConfirm` 确认对话框

### 版本控制规范
- 编写页面模拟数据时要区分开发版本和演示版本
- 参考 `docs/VERSION_CONTROL.md` 进行版本配置
- 演示版本只有数据治理模块使用真实接口，其他模块使用模拟数据

### 代码维护规范
- 修改模块时检查代码，删除未使用的引用和变量
- 生成或更新文件时，注释等内容要求中文输出，禁止乱码
- 优先查看网络上的业务实现作为参考

## 业务理解要求
作为资深前端架构师，需要深入理解医学数据治理业务场景：
- 数据治理工作流的配置和执行
- 数据质控的多维度检查（可靠性、及时性、完整性、一致性、准确性）
- 医学数据的标注和解析
- 数据检索和可视化分析
- 基于 RBAC 的权限管理

## 文件引用支持
可以通过 `#[[file:<relative_file_name>]]` 引用其他文件内容，如：
- OpenAPI 规范文件
- GraphQL Schema 文件
- 配置文件等
